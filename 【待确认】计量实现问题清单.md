## #1 流水表冗余存储Token字段

**现象**：

流水表 `t_meter_tenant_user_statement` 中存了 `token_count`、`input_tokens`、`output_tokens`、`cached_tokens` 四个字段，但计量表 `t_meter_tenant_user_llm_metering` 已经完整记录了这些Token信息。

两张表通过 `metering_id` 关联。

**问题**：

- 计量表负责记录"消耗了多少Token"
- 流水表负责记录"点数余额怎么变化的"
- 职责应该分开，流水表不需要重复存Token

**影响**：

- 流水表按月分表，数据量大，冗余字段增加写入开销和存储量
- 两份Token数据如果不一致会造成混乱

**建议**：

流水表去掉Token相关字段，需要追溯时通过 `metering_id` JOIN 计量表即可。

**待确认**：是有意为之（查询便利/历史遗留）还是可以优化掉？

## #2 流水表缺少 balance_after（变化后余额）

**现象**：

需求文档中定义流水需要记录"扣完后余额多少"（balance_after），但当前实现中流水表没有这个字段。

**当前结构**：
- ✅ value — 变化点数（负=扣，正=加）
- ✅ stmtType — 扣点来源（SEAT/CREDIT/OVERDRAFT）
- ✅ createTime — 时间
- ❌ balance_after — 变化后余额（**缺失**）

**影响**：

- 查某一时刻的余额需要 SUM 全部历史流水，而不是直接读取
- 对账、审计时不够直观

**待确认**：是否需要补上？还是通过 SUM 流水计算余额就够用？

## #5 音色设计/复刻的"延迟计费"触发时机

**来源**：MiniMax 计费规则

**现象**：

MiniMax音色设计和快速复刻的规则为"调用接口获得音色时不收费，首次使用该音色进行语音合成时才收费"。

**问题**：

我们的计量模式是"调用完成后立即计量扣点"，但该服务的费用应在**首次使用音色合成时**触发，而非调用设计/复刻接口时。

**影响**：

如果后续接入MiniMax音色相关服务，Console计量接口需要能识别"首次使用某音色"并触发一笔一次性费用。公式配置本身不需要改（仍是T1按次），但触发逻辑需要特殊处理。

**建议**：作为扩展场景记录，当前不影响主流程设计。实际接入时再细化。

**找谁确认**：研发 + 业务方

## #6 MAAS返回数据是否支持多模态Token分类

**来源**：Gemini多模态计价需求

**问题**：

Gemini等多模态模型按文本/图片/视频/音频分别定价。要精确计费，MAAS返回的数据需要区分各模态的Token数。

**需要确认**：

2. 如果没有，能否扩展？

**影响**：决定多模态系数是否可启用

**找谁确认**：MAAS团队

## #7 Gemini上下文缓存存储费的处理方式

**来源**：Gemini计价规则

**现象**：

Gemini的上下文缓存除了"命中时折扣"外，还有**持续存储费**：$1~4.5/百万token/小时。也就是创建了缓存后，即使不用，每小时也有费用。

**问题**：

1. 这个存储费是否需要向客户转嫁？
2. 如果转嫁，需要新增一种计费维度（按时长×缓存大小）
3. 如果平台承担，如何监控成本？

**建议**：当前阶段先不处理，后续根据实际缓存使用情况决定。

**找谁确认**：产品+商业

## #8 Batch模式（Batch File / Batch Chat）是否有业务场景

**来源**：阿里Qwen计价规则

**现象**：

Qwen提供Batch File（异步文件处理，半价）和 Batch Chat（批量对话）两种Batch模式，价格低于标准模式。

**处理方式**：将Batch视为独立的"模型版本"配置（如 `qwen3.7-plus / BATCH_FILE`），不需要新增字段。

**待确认**：我们的业务场景是否会用Batch模式调用Qwen或其他模型？如果不用则无需配置。

**找谁确认**：研发 + MAAS团队

## #9 隐式缓存 vs 显式缓存的区分

**来源**：阿里Qwen计价规则

**现象**：

Qwen同时存在两种缓存机制且价格不同：
- 隐式缓存命中：0.40元/百万tokens（自动触发）
- 显式缓存命中：0.20元/百万tokens（主动创建）

当前方案只有一个"文本缓存命中系数"，无法同时覆盖两种价格。

**当前处理**：按显式缓存命中价格配置（更低），差额由平台承担。

**长期方案**：如果MAAS能区分返回两种缓存的Token数，可加字段。暂不实现。

**找谁确认**：MAAS团队

2. 如果后续接入海外厂商且时段定义用UTC或其他时区，再加这个字段？

**找谁确认**：产品 + MAAS团队
