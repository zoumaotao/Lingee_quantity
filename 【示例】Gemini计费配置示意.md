# 【示例】Gemini 全系列计费配置示意

> **用途**：验证升级后的方案能否覆盖 Gemini 全系列复杂计价模式  
> **结论**：✅ Token类全覆盖，搜索接地走非LLM公式表

---

## 一、Gemini 计价特点总结

Gemini是目前计价维度最复杂的厂商：

| 特点 | 说明 |
|------|------|
| 输入按模态分价 | 文本/图片/视频同价 或 音频单独贵 |
| 输出按模态分价 | 文本、音频、图片、视频各有单价 |
| 缓存按模态分价 | 文本/图片/视频同价，音频单独 |
| 上下文分段 | Pro系列：≤20万token / >20万token |
| 思考Token | 包含在输出价格中（不单独计价） |
| 图片输出按Token计 | 不同分辨率消耗不同Token数，但单价统一 |
| 搜索接地 | 按查询次数额外收费 |
| 缓存存储费 | 按小时收费（待确认是否转嫁） |

## 五、关键设计验证

### 图片输出为什么不走非LLM表？

Gemini的图片生成虽然产出是图片，但**计价单位仍然是Token**（$60/M output tokens）。不同分辨率只是消耗的Token数不同：
- 1K图: 1120 tokens
- 4K图: 2520 tokens

所以只需要在 `sys_ratio` 里配 `image_output_ratio = 60.00`，MAAS返回图片output tokens数，系统自动计算。不需要按张配置。

### 降级逻辑验证

| 模型类型 | 需要填的字段 | 其余 |
|---------|------------|------|
| 纯文本模型（GLM等） | 文本输入 + 文本输出 | 全部留空降级 |
| 音频加价模型（大部分Gemini） | 文本输入 + 音频输入 + 文本输出 | 图片/视频留空→降级到文本 |
| 全模态模型（Flash Live） | 全部填写 | 无需降级 |
| 图片输出模型 | 文本输入 + 文本输出 + 图片输出 | 其余降级 |

**结论**：降级规则保证了简单模型配2个字段就够、复杂模型按需补充，不存在过度配置的问题。
