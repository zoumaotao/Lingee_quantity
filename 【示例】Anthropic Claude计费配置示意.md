# 【示例】Anthropic Claude 全系列计费配置示意

> **用途**：验证方案能否覆盖 Anthropic Claude 全系列计价模式  
> **结论**：✅ 全部覆盖

---

## 一、Claude 计价维度总结

| 维度 | 说明 | 是否存在 |
|------|------|---------|
| 输入Token | 标准输入价 | ✅ 全部模型 |
| 输出Token | 标准输出价 | ✅ 全部模型 |
| 缓存写入（Prompt Caching Write） | 首次写入缓存，比标准输入贵（约1.25倍） | ✅ 全部模型 |
| 缓存读取（Prompt Caching Read） | 命中缓存，大幅折扣（约10%输入价） | ✅ 全部模型 |
| 推理Token | Extended Thinking，按输出价计 | 包含在输出价中，不单独计价 |
| 多模态分价 | ❌ 不分模态 | 无 |
| 上下文分段 | ❌ | 无 |
| 峰谷时段 | ❌ | 无 |
| 限时促销 | Sonnet 5 有限时定价 | 按原价配 |

**Anthropic的特点**：计价结构非常统一——所有模型都是"输入 + 输出 + 缓存写入 + 缓存读取"四维，只是价格不同。

## 三、Token类配置（t_meter_sys_ratio）

| 模型标识 | 版本 | 阈值 | 时段 | 文本输入 | 文本输出 | 文本缓存命中 | 缓存写入 | 推理输出 |
|---------|------|------|------|---------|---------|------------|---------|---------|
| claude-fable-5 | - | - | - | 10.00 | 50.00 | 1.00 | 12.50 | - |
| claude-opus-4.8 | - | - | - | 5.00 | 25.00 | 0.50 | 6.25 | - |
| claude-sonnet-5 | - | - | - | 3.00 | 15.00 | 0.20 | 2.50 | - |
| claude-haiku-4.5 | - | - | - | 1.00 | 5.00 | 0.10 | 1.25 | - |

> 说明：
> - 文本输入系数 = 标准输入价
> - 文本输出系数 = 标准输出价
> - 文本缓存命中系数 = Prompt Caching Read价
> - 缓存写入系数 = Prompt Caching Write价
> - 推理输出系数留空 → 降级到文本输出系数（Extended Thinking按输出价计）
> - 多模态系数全部留空 → 降级到文本系数（Claude不按模态分价）

## 四、覆盖验证

| 计价维度 | Claude定价 | 我们的配置方式 | 覆盖 |
|---------|-----------|-------------|------|
| 标准输入 | $1~10/MTok | 文本输入系数 | ✅ |
| 标准输出 | $5~50/MTok | 文本输出系数 | ✅ |
| 缓存写入 | $1.25~12.50/MTok | 缓存写入系数 | ✅ |
| 缓存读取 | $0.10~1.00/MTok | 文本缓存命中系数 | ✅ |
| Extended Thinking | 包含在输出价中 | 推理输出系数留空→降级到文本输出 | ✅ |
| 多模态 | 不分模态 | 模态系数留空→降级到文本 | ✅ |
| 限时促销（Sonnet 5） | $2/$10至8月底 | 按标准价$3/$15配置 | ✅ |

**结论**：✅ 完全覆盖。Claude是"缓存写入系数"这个字段最典型的使用场景。

---

## 五、配置特点

- **每个模型只需1行配置**：无分段、无峰谷、无多模态分价
- **全部4个系数都用到**：输入 + 输出 + 缓存命中 + 缓存写入
- **缓存写入 > 标准输入**：这是Anthropic的特点（写入缓存的Token比普通输入贵25%）
- **缓存读取极低**：约为标准输入价的10%（鼓励使用缓存）
- **系数规律清晰**：每个模型的系数成比例（Fable=2×Opus=Sonnet的标准价×3.33=Haiku的10倍）

---

## 六、计算示例

**场景**：调用 Claude Opus 4.8，输入50000 tokens（其中30000命中缓存、5000首次写入缓存），输出20000 tokens

```
points = (
    纯文本输入Token × 文本输入系数
  + 缓存命中Token × 文本缓存命中系数
  + 缓存写入Token × 缓存写入系数
  + 文本输出Token × 文本输出系数
) ÷ 换算基数

= (
    15000 × 5.00        （50000-30000-5000=15000纯输入）
  + 30000 × 0.50        （缓存命中）
  + 5000 × 6.25         （缓存写入）
  + 20000 × 25.00       （输出）
) ÷ 10000

= (75000 + 15000 + 31250 + 500000) ÷ 10000
= 621250 ÷ 10000
= 62.13 点
```
