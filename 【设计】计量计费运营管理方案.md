# 【设计】计量计费运营管理方案

> **版本**：v1.0  
> **编制人**：邹茂桃  
> **时间**：2026-07-06  
> **目标**：保障计费配置的高效运维，快速响应厂商调价、支持批量操作和跨环境管理

---

## 一、为什么需要运营管理

### 1.1 核心场景

| 场景 | 频率 | 紧迫程度 | 说明 |
|------|------|---------|------|
| 厂商调价 | 不定期（月/季） | 高 | DeepSeek 7月中旬峰谷、Qwen限时折扣到期等 |
| 新模型上线 | 频繁（周/月） | 中 | 接入新模型需要快速配好系数 |
| 批量调整 | 不定期 | 中 | 如"某厂商全系列统一涨价20%" |
| 多环境同步 | 每次配置变更后 | 中 | 测试环境验证后推到生产 |
| 配置审计 | 持续 | 低 | 查看谁改了什么、什么时候改的 |

### 1.2 当前痛点

- 修改系数后立即生效，无法提前配好定时切换
- 无法批量调整，每个模型逐个改
- 跨环境需要手动重复操作
- 厂商调价后需要人工发现，容易遗漏

## 二、功能清单

### 2.1 定时生效

| 项 | 说明 |
|----|------|
| 功能 | 配置新的系数/公式后，设置一个未来时间点生效，到时间自动切换 |
| 场景 | DeepSeek 7月15日开始峰谷定价 → 提前配好peak/standard两行，设置7月15日0:00生效 |
| 规则 | 一条配置同一时间只能有一个"待生效"版本排队 |
| 展示 | 列表中标注"⏰ 2026-07-15 00:00 生效" |
| 取消 | 未到生效时间前可取消 |

### 2.2 批量调整

| 项 | 说明 |
|----|------|
| 功能 | 选中多条配置，统一执行调整操作 |
| 支持的操作 | ① 倍率调整（所有系数×N）<br>② 统一设值（某个字段统一改为X）<br>③ 批量启用/停用 |
| 场景 | DeepSeek峰时=平时×2 → 选中所有DeepSeek standard行，批量×2生成peak行 |
| 确认 | 操作前展示影响范围（涉及N条配置），确认后执行 |

### 2.3 导入导出

| 项 | 说明 |
|----|------|
| 导出格式 | CSV 和 JSON 两种 |
| 导出范围 | 全部 / 按厂商筛选 / 按计费类型筛选 |
| 导入模式 | ① 覆盖（按唯一键匹配，存在则更新）<br>② 新增（只导入不存在的）<br>③ 全量替换（清空后重新导入） |
| 校验 | 导入前校验格式、必填项、唯一键冲突，展示校验结果 |
| 预览 | 导入前预览变更（新增N条、更新N条、删除N条） |

### 2.4 跨环境迁移

| 项 | 说明 |
|----|------|
| 功能 | 将一个环境（如测试）的配置同步到另一个环境（如生产） |
| 操作 | 选择源环境 → 选择目标环境 → 预览差异 → 确认同步 |
| 差异展示 | 哪些是新增的、哪些值不同、哪些目标环境有但源没有 |
| 安全 | 同步前需二次确认，同步后记录操作日志 |

### 2.5 变更对比

| 项 | 说明 |
|----|------|
| 功能 | 任何修改操作，执行前展示新旧值对比 |
| 展示方式 | 字段级diff（旧值 → 新值），变化的字段高亮 |
| 适用范围 | 单条编辑、批量调整、导入覆盖、跨环境同步 |

### 2.6 模板复制

| 项 | 说明 |
|----|------|
| 功能 | 从现有模型的配置快速复制到新模型 |
| 操作 | 选择源模型 → 填写新模型标识 → 可选调整系数 → 创建 |
| 场景 | 新接入一个和GLM-5.1价格相近的模型，直接复制GLM配置再微调 |

### 2.7 版本历史与回滚

| 项 | 说明 |
|----|------|
| 功能 | 每次变更自动保存为新版本（不覆盖旧记录），保证可审计 |
| 规则 | 配置的每一次修改都生成新版本记录，旧版本永久保留不删除 |
| 记录内容 | 变更时间、操作人、变更前/后完整配置、变更原因（可选备注） |
| 回滚 | 选择某个历史版本，基于该版本创建新的生效版本（不是删除后续版本） |
| 已确认 | 此机制已与开发确认，配置更新均保存新版本，保证可审计 |

### 2.8 厂商调价巡检告警（后续）

| 项 | 说明 |
|----|------|
| 功能 | 定时抓取厂商官方定价页面，发现变化时自动告警 |
| 告警方式 | 站内通知 + 邮件/钉钉 |
| 状态 | 后续实现（当前先人工监控） |

## 三、操作流程

### 3.1 厂商调价响应流程

```
① 发现调价（人工监控 / 后续自动巡检告警）
    ↓
② 确认新价格，计算新系数
    ↓
③ 在管理页面配置新值
    ├─ 立即生效：直接保存
    └─ 定时生效：设置生效时间
    ↓
④ 变更对比确认
    ↓
⑤ 测试环境验证
    ↓
⑥ 跨环境迁移到生产
    ↓
⑦ 自动记录版本历史
```

### 3.2 新模型上线流程

```
① 确认厂商定价信息
    ↓
② 选择最相似的现有模型 → 模板复制
    ↓
③ 调整系数值
    ↓
④ 预览计算验证（输入模拟Token，查看计算出的点数是否合理）
    ↓
⑤ 保存 → 测试环境验证 → 同步到生产
```

### 3.3 批量调价流程

```
① 确认调价范围（如"DeepSeek全系列峰时×2"）
    ↓
② 筛选相关配置行
    ↓
③ 批量调整（倍率×2）
    ├─ 生成新行（peak）保留旧行（standard）
    └─ 或直接修改现有行系数
    ↓
④ 变更对比确认
    ↓
⑤ 设置生效时间（如7月15日）
    ↓
⑥ 自动到时切换
```

## 四、页面设计

### 4.1 主要入口

运营管理功能集成在现有的"大模型系数管理"和"计费公式管理"两个页面中，不单独开页面。通过工具栏按钮和操作菜单提供入口：

| 入口 | 位置 | 功能 |
|------|------|------|
| 导入 | 页面顶部工具栏 | 批量导入配置 |
| 导出 | 页面顶部工具栏 | 导出当前配置 |
| 批量调整 | 选中多行后出现 | 倍率调整/统一设值/批量启停 |
| 跨环境同步 | 页面顶部工具栏 | 同步到其他环境 |
| 模板复制 | 单个模型的操作菜单 | 复制到新模型 |
| 定时生效 | 编辑弹窗/创建弹窗中 | 设置生效时间 |
| 变更历史 | 单个模型的详情面板 | 查看版本记录 |

### 4.2 定时生效交互

在编辑/创建配置时，保存按钮旁增加"定时生效"选项：

```
┌─────────────────────────────────┐
│  保存方式：                      │
│  ○ 立即生效                     │
│  ○ 定时生效                     │
│     生效时间：[2026-07-15 00:00] │
│     备注：DeepSeek峰谷定价生效   │
└─────────────────────────────────┘
```

### 4.3 批量调整交互

选中多条后，顶部出现批量操作栏：

```
┌─────────────────────────────────────────┐
│ 已选中 4 条  │ 倍率调整 │ 统一设值 │ 取消 │
└─────────────────────────────────────────┘

倍率调整弹窗：
┌─────────────────────────────────────────┐
│ 批量倍率调整                             │
│                                         │
│ 影响范围：4条配置                        │
│ 调整倍率：[ 2.0 ]                       │
│ 适用字段：☑文本输入 ☑文本输出 ☑缓存命中   │
│           ☑缓存写入 ☑推理输出            │
│                                         │
│ 预览：                                  │
│ deepseek-v4-flash: 1.00→2.00 / 2.00→4.00│
│ deepseek-v4-pro:   3.00→6.00 / 6.00→12.0│
│                                         │
│              [取消]  [确认调整]           │
└─────────────────────────────────────────┘
```

### 4.4 跨环境同步交互

```
┌─────────────────────────────────────────┐
│ 跨环境同步                               │
│                                         │
│ 源环境：[测试环境 ▼]                     │
│ 目标环境：[生产环境 ▼]                   │
│                                         │
│ 差异预览：                               │
│ ┌─────────────────────────────────────┐ │
│ │ + 新增 2 条                         │ │
│ │   minimax-m3 (CHAT, ≤512K)         │ │
│ │   minimax-m3 (CHAT, >512K)         │ │
│ │ ~ 变更 1 条                         │ │
│ │   deepseek-v4-pro: 输入 3.0→6.0    │ │
│ │ - 目标多出 0 条                     │ │
│ └─────────────────────────────────────┘ │
│                                         │
│              [取消]  [确认同步]           │
└─────────────────────────────────────────┘
```

## 五、与现有系统的关系

| 功能 | 影响的系统 | 说明 |
|------|-----------|------|
| 定时生效 | kdmanage-backend | 需要定时任务扫描待生效配置，到时间后激活并推送 |
| 批量调整 | kdmanage-backend/frontend | 后端支持批量更新API，前端多选+操作栏 |
| 导入导出 | kdmanage-backend/frontend | 后端CSV/JSON解析+导出API，前端文件上传+下载 |
| 跨环境同步 | kdmanage-backend | 需要访问多个tmanage集群的配置 |
| 版本历史 | kdmanage-backend | 每次变更写入历史表（已有quota_ratio_history基础） |

## 六、上线建议

| 阶段 | 功能 | 说明 |
|------|------|------|
| Phase 1 | 导入导出 + 变更对比 | 最基础的批量操作能力 |
| Phase 2 | 定时生效 | 应对DeepSeek峰谷等定时调价 |
| Phase 3 | 批量调整 + 模板复制 | 提升日常运维效率 |
| Phase 4 | 跨环境同步 | 多环境协同 |
| Phase 5 | 厂商调价巡检告警 | 后续自动化 |
