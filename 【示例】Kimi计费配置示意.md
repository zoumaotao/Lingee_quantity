# 【示例】月之暗面 Kimi 全系列计费配置示意

> **用途**：验证方案能否覆盖 Kimi 全系列计价模式  
> **结论**：✅ 全部覆盖

---

## 一、Kimi 计价维度总结

| 维度 | 说明 | 是否存在 |
|------|------|---------|
| 输入Token（缓存未命中） | 标准输入价 | ✅ 全部模型 |
| 输出Token | 标准输出价 | ✅ 全部模型 |
| 缓存命中输入 | 命中缓存的折扣输入 | ✅ K2系列有 |
| 缓存写入 | ❌ | 无 |
| 推理Token | ❌（包含在输出价中） | 无 |
| 上下文分段 | ❌ K2系列无分段；Moonshot V1按模型名分段 | 见下方说明 |
| 峰谷时段 | ❌ | 无 |
| 多模态分价 | ❌（多模态模型输入统一价） | 无 |

**特殊点**：Moonshot V1系列按上下文窗口大小分为不同模型（8k/32k/128k），每个模型价格不同。这不是"同一模型分段"，而是**不同的模型ID**。

## 三、覆盖验证

| Kimi模型 | 计价方式 | 配置方式 | 覆盖 |
|---------|---------|---------|------|
| K2.7 Code | 输入+输出+缓存命中 | 3个系数 | ✅ |
| K2.7 Code Highspeed | 输入+输出+缓存命中（2倍） | 独立模型行 | ✅ |
| K2.6 多模态 | 输入+输出+缓存命中（多模态同价） | 3个系数 | ✅ |
| K2.5 多模态 | 输入+输出+缓存命中 | 3个系数 | ✅ |
| Moonshot V1-8k/32k/128k | 不同模型不同价 | 每个模型独立一行 | ✅ |
| Vision Preview | 与同窗口文本模型同价 | 独立模型行 | ✅ |

**结论**：✅ 完全覆盖，无需任何方案调整。

---

## 四、配置特点

- **Kimi最简单**：只用到3个系数（文本输入+文本输出+文本缓存命中），不需要多模态分价、分段、峰谷
- **Moonshot V1的"分段"**：不是真正的分段（同一模型按上下文长度不同价），而是厂商直接给了不同模型名。我们直接按不同模型配置即可
- **Highspeed版本**：价格翻倍，独立一行配置
- **多模态不加价**：K2.6/K2.5虽然支持图片输入，但不额外收费，统一走文本输入系数
