# 【示例】阿里 Qwen 全系列计费配置示意

> **用途**：验证方案能否覆盖阿里Qwen的全部计价模式  
> **结论**：✅ 全部覆盖

---

## 一、Qwen 计价维度总结

| 维度 | 说明 | 覆盖方式 |
|------|------|---------|
| 输入Token | 标准输入 | 文本输入系数 |
| 输出Token | 标准输出 | 文本输出系数 |
| 缓存命中 | 隐式缓存命中（自动） | 文本缓存命中系数 |
| 显式缓存创建 | 主动创建缓存，比标准输入贵 | 缓存写入系数 |
| 显式缓存命中 | 命中主动创建的缓存 | 文本缓存命中系数 |
| Batch File | 异步文件处理，半价 | 独立模型版本（BATCH_FILE） |
| Batch Chat | 批量对话 | 独立模型版本（BATCH_CHAT） |
| 上下文分段 | qwen3.6-plus: ≤256K vs 1M | 上下文阈值 |
| 工具调用（按次） | web_search等 | 非LLM公式表（T1） |
| ASR | 按秒 | 非LLM公式表（T2） |
| TTS | 按万字符 | 非LLM公式表（T3） |
| 图片生成 | 按张 | 非LLM公式表（T4） |
| 视频生成 | 按秒×分辨率 | 非LLM公式表（T2按时长，每档一行） |

## 二、Token类配置（t_meter_sys_ratio）

### 2.1 qwen3.7-plus（标准模式）

| 模型标识 | 版本 | 阈值 | 时段 | 文本输入 | 文本输出 | 文本缓存命中 | 缓存写入 |
|---------|------|------|------|---------|---------|------------|---------|
| qwen3.7-plus | CHAT | - | - | 2.00 | 8.00 | 0.40 | 2.50 |

> 说明：按原价配置（不配促销折扣价）。显式缓存命中=0.20，但由于比隐式缓存命中(0.40)更低，配0.20到文本缓存命中系数。

⚠️ **问题**：Qwen同时存在"隐式缓存命中"(0.40)和"显式缓存命中"(0.20)两个价格。当前方案只有一个"文本缓存命中系数"。

**处理方式**：配置显式缓存命中价格（0.20），因为显式缓存命中更精确。隐式缓存命中的差异由平台承担（厂商返回的是哪种缓存，需要MAAS区分）。

## 五、发现的问题

### 隐式缓存 vs 显式缓存

Qwen同时存在两种缓存机制，价格不同：
- 隐式缓存命中：0.40元/百万tokens（qwen3.7-plus）
- 显式缓存命中：0.20元/百万tokens

当前方案只有一个"文本缓存命中系数"，无法同时覆盖两种缓存价格。

**当前处理**：配显式缓存命中的价格（更低），隐式缓存命中的差额由平台承担。

**长期方案**：如果MAAS能区分返回"隐式缓存命中Token"和"显式缓存命中Token"，可以在sys_ratio中再加一个字段。暂不实现。
