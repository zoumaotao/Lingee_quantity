## 一、整体流程（一句话版）

```
定时任务触发 → 从MAAS拉取Token消耗 → Token×系数=点数 → 先扣席位→再扣共享池 → 写入流水 → 更新缓存
```

## 二、计量（Metering）

### 定义

**计量 = 记录每次LLM调用消耗了多少Token**

### 数据来源

不是业务系统主动上报，而是**定时任务从MAAS拉取**：

```java
// MeteringDeductionServiceImpl.runOnce()
List<MaasUsageDetail> details = maasUsageClient.fetchDetails(start, effectiveEnd, maasTenantIds);
```

- 拉取方式：定时任务（MeteringDeductionTask）
- 拉取源：MAAS管理端 `/manage/api/admin/usage/details`
- 拉取窗口：上次游标 ~ 当前时间（最长5分钟窗口）
- 游标管理：`t_pull_cursor` 表，记录上次拉取到哪里了

### MAAS返回的一条用量明细（MaasUsageDetail）

| 字段 | 说明 |
|------|------|
| tenantId | 租户ID |
| userId | 用户ID |
| modelType | 模型类型（映射为 llmId） |
| modelId | 模型ID（映射为 llmVersion） |
| inputTokens | 输入Token数 |
| outputTokens | 输出Token数 |
| cachedTokens | 缓存Token数（已缓存的输入Token） |
| requestId | 请求唯一标识（幂等） |
| requestTime | 请求时间 |
| agent | 智能体ID |
| skill | 技能ID |

### 计量记录落库

## 三、Token → 点数计算（已实现 ✅）

### 核心方法

`MeteringDeductionServiceImpl.getBigDecimal()`

### 计算公式

```
points = (inputTokens - cachedInputTokens) × inputRatio
       + (outputTokens - cachedOutputTokens) × outputRatio
       + cachedInputTokens × cacheInputRatio
       + cachedOutputTokens × cacheOutputRatio
       ÷ conversionRate（默认10000）
```

### 代码实现

```java
private BigDecimal getBigDecimal(MaasUsageDetail d, SysRatio ratio, int pointScale, long conversionRate) {
    // 获取系数（优先用独立字段，降级到ratio字段，再降级到配置默认值）
    BigDecimal inputRatio = ratio.getInputRatio() != null ? ratio.getInputRatio()
            : (ratio.getRatio() != null ? ratio.getRatio() : defaultInput);
    BigDecimal outputRatio = ratio.getOutputRatio() != null ? ratio.getOutputRatio()
            : (ratio.getRatio() != null ? ratio.getRatio() : defaultOutput);
    BigDecimal cacheInputRatio = ratio.getCacheInputRatio() != null ? ratio.getCacheInputRatio()
            : (ratio.getRatio() != null ? ratio.getRatio() : defaultCacheInput);
    BigDecimal cacheOutputRatio = ratio.getCacheOutputRatio() != null ? ratio.getCacheOutputRatio()
            : (ratio.getRatio() != null ? ratio.getRatio() : defaultCacheOutput);

    // 防止缓存数值大于总量时出现负数
    long pureInputTokens = Math.max(0L, inputTokens - cachedInputTokens);
    long pureOutputTokens = Math.max(0L, outputTokens - cachedOutputTokens);

    // 计算
    BigDecimal rate = BigDecimal.valueOf(conversionRate); // 默认10000
    return BigDecimal.valueOf(pureInputTokens).multiply(inputRatio)
            .add(BigDecimal.valueOf(pureOutputTokens).multiply(outputRatio))
            .add(BigDecimal.valueOf(cachedInputTokens).multiply(cacheInputRatio))
            .add(BigDecimal.valueOf(cachedOutputTokens).multiply(cacheOutputRatio))
            .divide(rate, pointScale, RoundingMode.UP); // 向上取整
}
```

### 系数来源

1. **SysRatioCache** 定时同步系数表 `t_meter_sys_ratio`
2. 系数由 **kdmanage-backend** 推送到 tmanage：  
   `POST /api/backend/metering/sys-ratio/push`
3. 查找逻辑：`sysRatioCache.find(llmId, llmVersion)`
4. 找不到系数时，使用 `buildDefaultRatio()` 兜底

### 计算示例

假设 glm-5 模型：inputRatio=1.0, outputRatio=3.6, cacheInputRatio=0.25

```
用户调用：inputTokens=50000, outputTokens=20000, cachedTokens=5000

pureInput  = 50000 - 5000 = 45000
pureOutput = 20000 - 0 = 20000

points = (45000 × 1.0 + 20000 × 3.6 + 5000 × 0.25 + 0 × 0) ÷ 10000
       = (45000 + 72000 + 1250) ÷ 10000
       = 118250 ÷ 10000
       = 11.825 → 11.83（向上取整到4位小数）
```

## 四、扣点（Deduction）

### 定义

**扣点 = 把计算出的点数，从用户的配额中扣减**

### 扣点执行器（DeductionExecutor.deductOne）

#### 扣减顺序（4步）

```
                    计算出的点数
                        │
           ┌────────────┴────────────┐
           ↓                         │
    ① 先扣席位配额                    │
    (item_type=0, 跟人走)             │
           │                         │
           │ 剩余点数                 │
           ↓                         │
    ② 累加到分组总额表                │ (group表只累加，不做额度限制)
    (t_meter_tenant_group_quota)     │
           │                         │
           │ 全部点数                 │
           ↓                         │
    ③ 累加到用户共享额度表            │ (user表只累加)
    (t_meter_tenant_user_quota)      │
           │                         │
           │ 剩余点数                 │
           ↓                         ↓
    ④ 循环扣共享池
    (item_type=1, 按deduction_order排序)
           │
           │ 如果还有剩余
           ↓
    ⑤ 生成透支(OVERDRAFT)流水
```

#### 代码实现

```java
public void deductOne(DeductionContext ctx, ...) {
    BigDecimal pointsRemaining = ctx.points;

    // ① 席位点数扣减（item_type = 0）
    TenantItemQuotaVO seat = pickFirstAvailable(seatMap.get(uid));
    if (seat != null && pointsRemaining > 0) {
        BigDecimal avail = seat.amount - seat.meter;  // 可用额度
        BigDecimal deduct = min(pointsRemaining, avail);
        if (deduct > 0) {
            seat.meter += deduct;                     // 更新已用
            stmts.add(buildStmt(ctx, uid, deduct, StatementType.SEAT));  // 记流水
            pointsRemaining -= deduct;
        }
    }

    // ② 按tenantId在group表上累加（允许 meter > amount）
    groups.forEach(g -> g.meter += pointsRemaining);

    // ③ 按userId在user表上累加share_meter
    user.shareMeter += pointsRemaining;

    // ④ 共享池循环扣减（item_type = 1, 按deduction_order排序）
    while (pointsRemaining > 0 && shared不为空) {
        TenantItemQuotaVO head = shared.peekFirst();
        // 检查有效期
        if (!isWithinTimeRange(head, ctx.requestTime)) {
            shared.pollFirst(); continue;
        }
        BigDecimal avail = head.amount - head.meter;
        if (avail <= 0) { shared.pollFirst(); continue; }
        
        BigDecimal deduct = min(pointsRemaining, avail);
        head.meter += deduct;
        stmts.add(buildStmt(ctx, uid, deduct, StatementType.CREDIT));
        pointsRemaining -= deduct;
    }

    // ⑤ 如果所有额度包耗尽仍有剩余 → 生成透支流水
    if (pointsRemaining > 0) {
        stmts.add(buildStmt(ctx, uid, pointsRemaining, StatementType.OVERDRAFT));
    }
}
```

#### 关键细节

| 项 | 说明 |
|----|------|
| **席位配额** | `amount - meter` = 可用额度，先扣这个 |
| **共享池** | 按 `deduction_order` 排序，逐个扣到用完 |
| **有效期** | 共享池有 startTime/endTime，超出范围的跳过 |
| **透支** | 所有池子都扣完了还不够 → OVERDRAFT 流水 |
| **Group累加** | group 表只做统计累加，不限制额度（允许 meter > amount） |
| **User累加** | user 表也只做统计累加（share_meter），用于仪表板展示 |

## 五、流水（Statement）

### 定义

**流水 = 每次配额变化的记录，不能修改不能删除**

### 流水类型（StatementType）

| 类型 | 含义 | value方向 | 触发时机 |
|------|------|----------|---------|
| **SEAT** | 席位配额扣减 | 负值 | 调用消耗，扣席位 |
| **CREDIT** | 共享池扣减 | 负值 | 调用消耗，扣共享池 |
| **OVERDRAFT** | 透支 | 负值 | 所有池子耗尽仍有消耗 |
| **SEAT_GRANT** | 席位额度发放 | 正值 | 首次使用席位时记录 |
| **CREDIT_ADD** | 共享池额度发放 | 正值 | 首次使用共享池时记录 |
| **CREDIT_GRANT** | 赠送点数 | 正值 | 配额过期重新发放 |
| **CREDIT_GRANT_INCREMENT** | 点数增加 | 正值 | 额度增加 |
| **CREDIT_EXPIRE** | 点数过期 | 负值 | 配额到期清零 |

### 流水记录结构

```java
TenantUserStatement {
    id              // 流水唯一ID（雪花）
    tenantId        // 租户ID
    userId          // 用户ID
    value           // 变化值（负=扣减，正=增加）
    stmtType        // 流水类型（SEAT/CREDIT/OVERDRAFT/...）
    tokenCount      // Token总数
    inputTokens     // 输入Token
    outputTokens    // 输出Token
    cachedTokens    // 缓存Token
    ratioVersion    // 系数版本号
    llmId           // 模型ID
    llmVersion      // 模型版本
    requestId       // MAAS请求ID（幂等）
    agentId         // 智能体ID
    skillId         // 技能ID
    meteringId      // 关联计量表ID
    meteringType    // 计费类型（LLM）
    createTime      // 创建时间（分表键，按月分表）
}
```

### 流水示例

**一次调用可能产生多条流水**：

```
用户调用GLM-5，消耗 11.83 点
席位可用额度：5 点
共享池可用额度：100 点

→ 产生流水：
  ① SEAT:    value=-5.00   (席位扣5点)
  ② CREDIT:  value=-6.83   (共享池扣6.83点)
  合计：-11.83 点
```

## 六、完整编排流程

```java
// MeteringDeductionServiceImpl.runOnce(targetEnd)
@Transactional
public boolean runOnce(Instant targetEnd) {

    // 0. 刷新系数缓存
    sysRatioCache.reload();
    
    // 0.1 读取拉取水位线游标
    PullCursor cursor = pullCursorMapper.selectByCursorKey(key);
    Instant start = cursor.lastPullEndMs;  // 上次结束 = 本次开始
    
    // 0.2 限制窗口最长5分钟
    Instant effectiveEnd = min(targetEnd, start + 5min);

    // 1. 从MAAS拉取用量明细
    List<MaasUsageDetail> details = maasUsageClient.fetchDetails(start, effectiveEnd, tenantIds);

    // 2. 计算点数 + 构建扣减上下文
    BuildResult build = buildContexts(details);
    //   内部：对每条detail → 查系数 → Token×系数÷10000 = 点数
    //   同时创建 TenantUserLlmMetering 计量记录

    // 3. 加载配额数据（4张表）
    Map<userId, UserQuota> userQuotaMap = loadUserQuotaMap(userIds);
    Map<userId, List<SeatItem>> seatMap = loadSeatMap(userIds);
    Map<tenantId, Deque<SharedItem>> sharedMap = loadSharedListMap(tenantIds);
    Map<tenantId, List<GroupQuota>> groupMap = loadGroupMap(tenantIds);

    // 4. 逐条扣减
    for (DeductionContext ctx : ctxs) {
        deductionExecutor.deductOne(ctx, userQuotaMap, seatMap, sharedMap, groupMap, stmts, ...);
    }

    // 5. 批量持久化（一个事务内）
    //    - 写入计量表 (t_meter_tenant_user_llm_metering)
    //    - 写入流水表 (t_meter_tenant_user_statement)
    //    - 更新配额表 (item_quota.meter, group_quota.meter, user_quota.share_meter)
    //    - 推进游标 (pull_cursor)
    persistChanges(...);

    // 6. 事务提交后 → 异步刷新Redis缓存
    scheduleRedisAfterCommit(redisSnapshots, seatMap);
}
```
