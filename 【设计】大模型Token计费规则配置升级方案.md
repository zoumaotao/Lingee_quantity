# 【设计】大模型Token计费规则配置升级方案

> **版本**：v1.0  
> **编制人**：邹茂桃  
> **时间**：2026-07-03  
> **目标**：覆盖国内外主流Token计价大模型的全部计费维度

---

## 一、为什么要升级

### 1.1 现状不足

当前系数配置只支持4个维度：输入系数、输出系数、缓存输入系数、缓存输出系数。

无法满足以下已接入模型的计价需求：
- **DeepSeek**：上下文超64K后单价翻倍
- **Anthropic**：缓存写入单独计价（高于标准输入价）
- **DeepSeek R1 / Kimi Thinking / Anthropic Extended Thinking**：推理Token单独计价
- **部分国内厂商**：峰谷时段差异化定价

### 1.2 已接入模型

kimi2.6、qwen3.6plus、qwen3.6max、qwen3 vl、glm5.1、deepseekv4pro、deepseekv4flash、doubao-seed-2-0-mini、doubao-seed-1-6-vision、doubao-seed-2-0-pro、gemini-3.5-flash、gemini-3.1-flash、Anthropic全系列

### 3.2 一个模型多条配置的含义

**大部分模型**：一行配置即可（无分段、无时段）。

**复杂模型**：同一模型因分段或时段不同，有多条配置，系统根据请求上下文自动匹配：

| 模型 | 需要几条配置 | 原因 |
|------|------------|------|
| glm-5.1 | 1条 | 无分段无时段 |
| qwen3.6-plus | 1条 | 无分段无时段 |
| minimax-m3 | 2条 | 512K分界两档 |
| deepseek-v4-flash | 2条 | 峰谷两个时段 |
| deepseek-v4-pro | 2条 | 峰谷两个时段 |
| minimax-m3 + 峰谷（假设） | 最多 2×2=4条 | 分段数 × 时段数 |

### 3.3 唯一标识

一条配置由以下组合唯一确定：

```
模型标识 + 上下文阈值 + 时段标识
```

## 四、计算规则

### 4.1 公式

```
消耗点数 = (
    文本输入Token × 文本输入系数
  + 图片输入Token × (图片输入系数 ?? 文本输入系数)
  + 视频输入Token × (视频输入系数 ?? 文本输入系数)
  + 音频输入Token × (音频输入系数 ?? 文本输入系数)
  + 文本缓存命中Token × (文本缓存命中系数 ?? 文本输入系数)
  + 音频缓存命中Token × (音频缓存命中系数 ?? 文本缓存命中系数 ?? 文本输入系数)
  + 缓存写入Token × (缓存写入系数 ?? 文本输入系数)
  + 文本输出Token × 文本输出系数
  + 图片输出Token × (图片输出系数 ?? 文本输出系数)
  + 视频输出Token × (视频输出系数 ?? 文本输出系数)
  + 音频输出Token × (音频输出系数 ?? 文本输出系数)
  + 推理Token × (推理输出系数 ?? 文本输出系数)
) ÷ 换算基数
```

其中 `??` = 为空则使用后面的值（链式降级）。

### 4.2 降级规则

| 字段 | 降级链 |
|------|--------|
| 图片/视频/音频输入系数 | 为空 → 文本输入系数 |
| 文本缓存命中系数 | 为空 → 文本输入系数 |
| 音频缓存命中系数 | 为空 → 文本缓存命中系数 → 文本输入系数 |
| 缓存写入系数 | 为空 → 文本输入系数 |
| 图片/视频/音频输出系数 | 为空 → 文本输出系数 |
| 推理输出系数 | 为空 → 文本输出系数 |

### 4.3 对纯文本模型的简化

大部分模型（GLM、DeepSeek、Qwen等）只需要填**2个必填字段**：
- 文本输入系数
- 文本输出系数

其余全部留空 → 自动降级 → 效果等同于之前的 input_ratio + output_ratio。

### 4.4 匹配规则

系统收到一条消耗记录后，按以下顺序匹配配置：

1. 精确匹配模型标识
2. 若配置中存在上下文阈值 → 根据本次总输入Token数判断落入哪一档
3. 若配置中存在时段 → 根据请求时间+时区，判断是否落入时段规则
4. 找不到匹配 → 使用系统默认配置兜底
5. 按MAAS返回的各模态Token数，分别乘对应模态系数，求和

### 4.5 MAAS需要返回的数据

| 字段 | 说明 | 必须 |
|------|------|------|
| text_input_tokens | 文本输入Token数 | 是 |
| image_input_tokens | 图片输入Token数 | 否（为空视为0） |
| video_input_tokens | 视频输入Token数 | 否 |
| audio_input_tokens | 音频输入Token数 | 否 |
| text_output_tokens | 文本输出Token数 | 是 |
| image_output_tokens | 图片输出Token数 | 否 |
| video_output_tokens | 视频输出Token数 | 否 |
| audio_output_tokens | 音频输出Token数 | 否 |
| cache_hit_tokens | 文本/图片/视频缓存命中Token | 否 |
| audio_cache_hit_tokens | 音频缓存命中Token | 否 |
| cache_write_tokens | 缓存写入Token | 否 |
| reasoning_tokens | 推理/思考Token | 否 |

## 五点五、配置交互流程

### 创建流程（3步向导）

| 步骤 | 内容 | 说明 |
|------|------|------|
| **第1步：选择模型** | 填写模型名称、ID、版本、厂商 | 基本信息 |
| **第2步：规则维度** | 选择是否分段（配阈值列表）+ 是否分时段（配时段名+规则） | 系统实时显示"预计生成N行配置" |
| **第3步：填写系数** | 按分段×时段组合展开矩阵，逐行填写12个系数 | 非必填字段留空自动降级 |

### 详情展示

每个"分段×时段"组合为一个卡片，卡片内按三类分组展示全部12个系数：
- 输入系数（文本/图片/视频/音频）
- 输出系数（文本/图片/视频/音频）
- 缓存&推理（文本缓存命中/音频缓存命中/缓存写入/推理输出）

### 即时试算

创建和编辑时均提供计算预览区域，输入模拟Token数实时计算点数，验证配置正确性。

## 六、对现有功能的影响

| 影响点 | 说明 |
|--------|------|
| 现有模型配置 | **不受影响**——新增维度全部可为空，为空时自动按现有逻辑降级 |
| 现有计算结果 | **不变**——降级规则保证空值走老路径 |
| kdmanage系数管理页面 | 需要扩展编辑表单，支持新增维度的配置 |
| 系数推送 | 推送内容需扩展，携带新维度字段 |
| 可渐进上线 | 先扩展配置能力，再逐步给模型补充新维度数据 |
