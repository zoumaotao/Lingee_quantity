# 【示例】DeepSeek 计费配置示意

> **用途**：验证现有设计方案能否完整覆盖 DeepSeek 的计价模式  
> **结论**：✅ 完全覆盖

---

## 一、DeepSeek 官方计价

| 模型 | 输入（缓存命中） | 输入（缓存未命中） | 输出 |
|------|----------------|-----------------|------|
| deepseek-v4-flash | 0.02元/百万tokens | 1元/百万tokens | 2元/百万tokens |
| deepseek-v4-pro | 0.025元/百万tokens | 3元/百万tokens | 6元/百万tokens |

**其他信息**：
- 两个模型都支持思考模式（thinking）和非思考模式
- 上下文长度：1M
- 输出长度：最大384K
- 预计7月中旬采用峰谷定价：高峰时段价格为平时2倍
- 高峰时段：北京时间每日 9:00~12:00 和 14:00~18:00

---

## 二、论证分析

### 计价维度拆解

| 维度 | DeepSeek是否有 | 我们的方案是否覆盖 |
|------|--------------|-----------------|
| 输入Token | ✅ | ✅ input_ratio |
| 输出Token | ✅ | ✅ output_ratio |
| 缓存命中输入 | ✅（价格远低于标准输入） | ✅ cache_hit_input_ratio |
| 缓存写入 | ❌（不区分，未命中=标准输入价） | ✅ 不配即按input_ratio降级 |
| 推理Token | ❌（思考模式输出价格与普通输出一致） | ✅ 不配即按output_ratio降级 |
| 上下文分段 | ❌（当前不分段，1M统一价） | ✅ 不配阈值即不分段 |
| 峰谷时段 | ✅（7月中旬起） | ✅ time_tier + 时段规则 |

**关键发现**：
- DeepSeek的"缓存未命中"就是标准输入价，不需要额外字段
- 思考模式的输出价格和普通输出一致（不像Anthropic单独计价），不需要reasoning_output_ratio
- 当前没有上下文分段（与之前假设不同，之前是旧版V3的64K分段，V4已取消）
- 7月中旬峰谷定价即将生效

## 四、计算示例

### 示例1：v4-flash，非高峰，无缓存命中

```
输入: 50000 tokens（全部未命中缓存）
输出: 20000 tokens

points = (50000 × 1.00 + 0 × 0.02 + 20000 × 2.00) ÷ 10000
       = (50000 + 0 + 40000) ÷ 10000
       = 9.0 点
```

### 示例2：v4-flash，非高峰，部分缓存命中

```
输入: 50000 tokens（其中40000命中缓存，10000未命中）
输出: 20000 tokens

纯输入 = 50000 - 40000 = 10000
points = (10000 × 1.00 + 40000 × 0.02 + 20000 × 2.00) ÷ 10000
       = (10000 + 800 + 40000) ÷ 10000
       = 5.08 点
```

### 示例3：v4-pro，高峰时段

```
输入: 50000 tokens（全部未命中）
输出: 20000 tokens
时间: 北京时间 10:30（落入 09:00-12:00 高峰段）

匹配 peak 行：
points = (50000 × 6.00 + 0 + 20000 × 12.00) ÷ 10000
       = (300000 + 240000) ÷ 10000
       = 54.0 点

对比 standard 行：
points = (50000 × 3.00 + 0 + 20000 × 6.00) ÷ 10000
       = (150000 + 120000) ÷ 10000
       = 27.0 点

峰时确实是平时的2倍 ✅
```

## 五、覆盖验证

| 计价维度 | DeepSeek情况 | 我们的配置方式 | 覆盖 |
|---------|-------------|-------------|------|
| 标准输入 | 1元/3元 per M tokens | input_ratio | ✅ |
| 标准输出 | 2元/6元 per M tokens | output_ratio | ✅ |
| 缓存命中 | 0.02元/0.025元（极低折扣） | cache_hit_input_ratio | ✅ |
| 缓存写入 | 无（=标准输入价） | 字段留空，降级到input_ratio | ✅ |
| 推理Token | 无独立定价（=标准输出价） | 字段留空，降级到output_ratio | ✅ |
| 上下文分段 | 无（V4取消分段） | 不配阈值 | ✅ |
| 峰谷时段 | 2倍，09:00-12:00,14:00-18:00 | time_tier + 时段规则 | ✅ |

**结论**：✅ 完全覆盖，无需任何方案调整。

## 六、升级操作建议

DeepSeek 7月中旬峰谷生效时的操作：
1. 将现有2行配置（flash + pro）分别拆为2行（standard + peak）
2. peak行系数 = standard行 × 2
3. peak行时段规则填 `09:00-12:00,14:00-18:00`
4. 时区填 `Asia/Shanghai`
5. standard行不填时段规则（匹配所有未命中peak的时间）
